# MOEX Team Calculator — Контекст проекта

## Что это

Веб-приложение «Калькулятор продуктовых команд» для Московской биржи. Пользователь описывает задачу ИТ-команды, приложение через AI генерирует два варианта команды (традиционный и AI-powered с vibecoding), сравнивает их и выводит dashboard.

Цель — помочь обосновать создание продуктовой команды перед наблюдательным советом: состав, стоимость, метрики, риски, экономия при использовании AI.

## Контекст организации

- **Московская биржа** — крупная финансовая некредитная организация
- 2400 сотрудников (1500 RUN, 900 CHANGE)
- AI внедряется хаотично, есть сопротивление ИБ. Развёрнута in-house модель QWEN
- ИТ преимущественно водопадный подход, продуктовый подход в процессе запуска

## Стек технологий

- **Фронтенд:** один файл `index.html`, чистый HTML + CSS + JavaScript, без фреймворков
- **AI API:** Groq (бесплатный tier), модель `llama-3.3-70b-versatile`, OpenAI-совместимый формат
- **Дизайн:** тёмная финтех-тема (Bloomberg-стиль), шрифты JetBrains Mono + Plus Jakarta Sans
- **Экспорт:** PDF через html2pdf.js
- **Токен:** хранится в `env.js` (не коммитится), поддержка URL-параметра `?key=`, fallback на ручной ввод

## Что реализовано

### Форма ввода
- Два поля: описание задачи + ценность для бизнеса
- API-ключ: автозагрузка из env.js → URL-параметр → ручной ввод
- Валидация полей

### Анимация загрузки
- Круговой спиннер + прогресс-бар
- 7 сменяющихся статусов каждые 2.5 сек
- Работает параллельно с API-вызовом

### API-интеграция
- POST на `https://api.groq.com/openai/v1/chat/completions`
- response_format: json_object
- Системный промпт с таблицей зарплат Москвы 2026 (финтех)
- Возвращает JSON с двумя вариантами команды
- Обработка ошибок: 401, 429, network, JSON parse

### Dashboard — два варианта side-by-side
1. **Описание команды** — карточка с бизнес-описанием
2. **Сравнение подходов:**
   - Две вкладки: 🔵 Традиционный vs 🟣 AI-Powered
   - Баннер экономии (% и сумма в год)
   - 3 KPI-карточки: размер, стоимость/мес, стоимость/год
3. **AI-инструменты** — какие конкретно инструменты использует AI-команда
4. **Состав команд** — две таблицы рядом + два bar chart долей
5. **Метрики эффективности** — 3 карточки с gauge-диаграммами (порог/цель)
6. **Реестр рисков** — карточки + матрица рисков 3×3

### Экспорт
- PDF (html2pdf.js, A4, тёмная тема сохраняется)
- Копирование JSON в буфер

## Что можно доделать

1. **Экспорт DOCX** — подключить библиотеку docx через CDN, формировать документ программно
2. **Сравнение нескольких расчётов** — сохранять результаты в массив, режим side-by-side
3. **Редактирование inline** — contenteditable ячейки в таблице ролей, автопересчёт итогов
4. **История расчётов** — выпадающий список предыдущих результатов
5. **Подключение нескольких AI** — Groq + Gemini + HuggingFace для сравнения ответов
6. **Адаптация под конкретные процессы Мосбиржи** — преднастроенные шаблоны задач
