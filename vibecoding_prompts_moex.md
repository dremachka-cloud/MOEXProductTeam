# Набор промптов для Vibecoding
## «Калькулятор продуктовой команды Московской биржи»

> **Контекст для AI-кодера:** Мы создаём веб-приложение для Московской биржи. Пользователь вводит словесное описание задачи и ценности продуктовой команды. Приложение отправляет запрос в бесплатные нейросети (Google Gemini, Groq, HuggingFace) и на основе их ответов формирует красивый dashboard с составом команды, бюджетом, метриками и рисками. Стек: React (JSX), Tailwind CSS, recharts для графиков. Результат должен экспортироваться в PDF/DOCX.

---

## ФАЗА 1: КАРКАС ПРИЛОЖЕНИЯ

### Промпт 1 — Структура проекта

```
Создай React-приложение (один файл App.jsx) со следующей структурой:

1. Шапка с логотипом "MOEX Team Calculator" в стиле Московской биржи (синий #003DA5, белый, серый)
2. Левая панель (30% ширины): форма ввода с двумя полями:
   - textarea "Описание задачи" (placeholder: "Опишите задачу команды: что нужно сделать, какие системы интегрировать, сколько процессов автоматизировать...")
   - textarea "Ценность для бизнеса" (placeholder: "Какую ценность принесёт эта команда: сокращение времени, экономия, автоматизация...")
   - Кнопка "Рассчитать команду" (синяя, крупная)
3. Правая панель (70% ширины): область для dashboard, пока пустая с текстом "Введите описание задачи и нажмите Рассчитать"
4. Футер: "Внутренний инструмент Департамента эффективности"

Используй Tailwind CSS. Дизайн строгий, корпоративный, как для финансовой организации. Шрифт Inter или системный sans-serif.
```

## ФАЗА 2: ИНТЕГРАЦИЯ С НЕЙРОСЕТЯМИ

### Промпт 3 — Сервисный слой для API нейросетей

```
Создай модуль apiService.js с тремя функциями для вызова бесплатных нейросетей. Каждая функция принимает промпт и возвращает JSON-ответ.

1. callGemini(prompt, apiKey) — Google Gemini API (gemini-2.0-flash)
   URL: https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key={apiKey}
   POST запрос с JSON body

2. callGroq(prompt, apiKey) — Groq API (llama-3.3-70b-versatile)
   URL: https://api.groq.com/openai/v1/chat/completions
   OpenAI-совместимый формат, response_format: { type: "json_object" }

3. callHuggingFace(prompt, apiKey) — HuggingFace Inference API
   URL: https://api-inference.huggingface.co/models/mistralai/Mistral-7B-Instruct-v0.3

Каждая функция:
- try/catch обёртка
- timeout 30 секунд
- При ошибке: { error: true, message: "описание", provider: "имя" }
- При успехе: парсит и возвращает JSON

Функция-оркестратор callAllProviders(prompt, keys):
- Вызывает все три параллельно через Promise.allSettled
- Возвращает массив результатов с пометкой провайдера
- Retry логика: при ошибке повтор 1 раз через 2 сек
```

### Промпт 4 — Формирование промпта для нейросетей

```
Создай модуль promptBuilder.js с функцией buildTeamPrompt(taskDescription, businessValue).

Функция формирует промпт, который заставляет нейросеть вернуть строго JSON:

{
  "summary": {
    "mission": "описание задач и ценностей на бизнес-языке, 3-5 предложений",
    "key_deliverables": ["список ключевых результатов"]
  },
  "team": {
    "total_headcount": число,
    "roles": [
      {
        "title": "название роли",
        "category": "management|development|qa|security|support",
        "count": число,
        "seniority": "senior|middle|junior|lead",
        "salary_net_monthly": число в рублях на руки,
        "salary_gross_monthly": число полная стоимость для компании,
        "justification": "зачем нужна роль"
      }
    ]
  },
  "budget": {
    "monthly_fot": число,
    "yearly_fot": число,
    "monthly_with_overhead": число,
    "yearly_with_overhead": число,
    "overhead_percent": 15
  },
  "metrics": [
    {
      "name": "название",
      "description": "что измеряет",
      "threshold_value": "пороговое значение",
      "target_value": "целевое значение",
      "measurement_method": "как измерять"
    }
  ],
  "risks": [
    {
      "title": "название риска",
      "probability": "high|medium|low",
      "impact": "high|medium|low",
      "mitigation": "как снижать",
      "category": "people|process|technology|budget|security"
    }
  ]
}

В промпте ОБЯЗАТЕЛЬНО:
- Контекст: "Ты CTO крупной российской финансовой организации (биржа, 2400 сотрудников)"
- "Зарплаты реальные для Москвы 2026 года в рублях"
- "Полная стоимость = зарплата на руки / 0.87 * 1.302 * 1.2"
- "Ответь ТОЛЬКО JSON, без markdown, без пояснений"
```

### Промпт 5 — Настройки API-ключей

```
Добавь панель настроек (шестерёнка в шапке). Модальное окно "Настройки API":

1. Три поля для ключей:
   - Google Gemini (ссылка https://aistudio.google.com/apikey)
   - Groq (ссылка https://console.groq.com)
   - HuggingFace (ссылка https://huggingface.co/settings/tokens)

2. "Проверить подключение" — тестовый запрос, статусы:
   - Зелёный: Подключено
   - Красный: Ошибка
   - Серый: Не настроено

3. Ключи в React state (НЕ localStorage)
4. Если ключей нет — toast: "Настройте хотя бы один API-ключ"
```

---

### Промпт 6 — Связка формы с API

```
Свяжи форму с реальными вызовами нейросетей:

1. При "Рассчитать команду":
   - Сформировать промпт через buildTeamPrompt()
   - Progress tracker привязан к реальным событиям:
     * "Запрос к Google Gemini..." / "Запрос к Groq..." / "Запрос к HuggingFace..."
     * Время ответа: "Groq ответил за 2.3 сек"

2. Обработка ответов:
   - Несколько ответов — выбрать самый полный
   - Один ответ — использовать его
   - Ни одного — сообщение об ошибке + "Попробовать снова"

3. Валидация:
   - JSON парсится? Обязательные поля есть?
   - Битый JSON — извлечь regex-ом (убрать markdown-обёртку)
```

### Промпт 9 — Блок "Бюджет"

```
Компонент BudgetDashboard (budget + roles):

1. Четыре KPI-карточки: ФОТ/мес, ФОТ/год, с накладными/мес, с накладными/год

3. Pie chart: доли ролей в бюджете

Числа через Intl.NumberFormat('ru-RU') с символом рубля.
```

### Промпт 10 — Блок "Метрики эффективности"

```
Компонент MetricsPanel (массив metrics):

1. Для каждой метрики — карточка:
   - Название, описание
   - Прогресс-бар с зонами: красная (0-порог), жёлтая (порог-цель), зелёная (цель+)

### Промпт 11 — Блок "Риски"

```
Компонент RiskMatrix (массив risks):

   - Бейджи вероятности и влияния

   - Стратегия митигации

### Промпт 16 — Обработка ошибок

```
Edge cases:
1. Пустые поля — валидация с красной рамкой
2. Короткий текст < 20 символов — предупреждение
3. Не-JSON ответ — извлечение regex
4. Неполный JSON — показать что есть, пометить остальное
5. Все API упали — детали ошибок + "Повторить"
6. Таймаут 30 сек
7. Rate limit 429 — "Подождите минуту"
```
